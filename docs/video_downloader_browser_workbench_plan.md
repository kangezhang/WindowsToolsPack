# 视频下载工具浏览器工作台改造方案

## 目标

把当前“左侧下载表单 + 右侧嵌入网页”的工具，升级为更像浏览器的视频下载工作台：

- 浏览网页是主体验，下载控制是围绕当前页面的辅助能力。
- 下载状态更清晰，日志不再占据主要空间。
- m3u8、B站、普通站点都走统一的“当前页面 -> 解析 -> 下载”工作流。
- 保留现有 yt-dlp、ffmpeg、安卓模式、m3u8 抓取和并发分片能力。

## 设计原则

- 第一阶段只调整 UI 信息架构，不重写下载核心，降低回归风险。
- 浏览器区域应占据最大空间，下载面板保持紧凑。
- 顶部工具栏负责浏览器行为，下载面板负责解析和任务创建。
- 日志和进度放到底部下载抽屉，默认可见但不抢主视图。
- 按钮文案短、位置固定，常用操作不隐藏到二级菜单。

## 目标布局

### 顶部浏览器栏

横向占满窗口顶部，包含：

- 后退
- 前进
- 刷新
- 主页
- 地址栏
- 浏览模式：桌面 / 安卓
- Chrome 打开

顶部栏只放浏览器导航相关功能，不放下载功能。

### 主内容区

使用水平分割布局：

- 左侧：下载控制面板，宽度约 360-460px。
- 右侧：嵌入浏览器，占主要空间。

左侧下载控制面板包含：

- 当前链接
- 粘贴
- 使用当前网页
- 抓取 m3u8
- 获取信息
- 下载范围
- 分辨率
- 并发分片
- 保存路径
- 开始下载 / 取消
- ffmpeg 状态

### 底部下载抽屉

位于窗口底部，包含：

- 当前视频信息
- 下载进度条
- 精简日志

后续阶段可升级为多任务列表。

## 分阶段实施

### 第一阶段：布局重构

范围：

- 把浏览器导航从右侧面板移动到窗口顶部。
- 右侧面板只保留 WebEngine 浏览器。
- 左侧下载面板保留下载输入和操作。
- 底部新增状态抽屉，移动信息、日志、进度条。
- 保持所有函数名和下载逻辑尽量不变。

验收：

- 窗口能正常打开。
- 浏览器导航可用。
- 当前网页 URL 同步到下载链接。
- 获取信息、抓取 m3u8、开始下载仍可触发。
- 编译通过。

### 第二阶段：任务列表

范围：

- 引入 DownloadTask 数据模型。
- 把单个进度和日志封装到任务。
- 底部抽屉升级为任务列表。
- 支持重试、取消、打开文件夹。

### 第三阶段：设置和持久化（已开始）

范围：

- 默认下载目录。
- 默认并发分片。
- 默认浏览模式。
- B站 vd_source。
- 常用请求头模板。

当前已实现：

- 使用 `SystemConfig.get_config_dir()` 下的 `video_downloader_settings.json` 保存设置。
- 启动时读取默认下载目录。
- 启动时读取默认并发分片。
- 启动时读取默认浏览模式。
- 启动时读取 B站 `vd_source`。
- 下载请求头支持从设置文件合并自定义 headers。
- 顶部工具栏提供“设置”按钮。
- 设置面板支持编辑默认下载目录、默认并发分片、默认浏览模式、B站 `vd_source` 和自定义请求头 JSON。
- 设置保存后立即应用到当前窗口。

### 第四阶段：浏览器内核增强（已开始）

范围：

- 评估从 Qt WebEngine 切换到 Edge WebView2。
- 保留 Qt 方案作为跨平台 fallback。
- 优先解决 B站内嵌播放兼容性。

当前已实现：

- 保留 Qt WebEngine 作为主工作台内嵌浏览器。
- 新增独立进程的 Edge WebView2 预览窗口。
- 顶部工具栏新增“WebView2打开”按钮，用 Edge WebView2 打开当前页面。
- WebView2 预览用于解决 Qt WebEngine 在 B站等站点里播放器一直加载或 HTML5 能力检测不通过的问题。

当前限制：

- WebView2 目前是独立预览窗口，不是嵌入右侧 Qt 面板。
- 深度嵌入 WebView2 到 PySide6 QWidget 需要额外 Win32/WinForms 承载层，复杂度高，后续单独评估。
